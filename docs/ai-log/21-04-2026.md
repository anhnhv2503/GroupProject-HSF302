## Phiên 10 — Tích Hợp Reviews Vào Product Detail

**Thời gian:** 20:58 — 22:00

### Câu hỏi của tôi

> Tích hợp phần đánh giá trực tiếp vào trang product-detail thay vì trang review riêng. Chỉ hiển thị đánh giá đã được duyệt (isVisible=true). Review chỉ được submit bởi users đã mua sản phẩm và có order DELIVERED.

### Câu trả lời của AI

AI đã:
1. Cập nhật `ProductClientController.java` để load reviews vào product detail
2. Thêm method `getVisibleReviewsByProduct()` vào service để chỉ lấy reviews đã duyệt
3. Thêm POST endpoint `/products/{id}/review` để submit review
4. Cập nhật `product-detail.html` hiển thị reviews tab
5. Xóa trang review riêng và ReviewController

### Tôi đã làm gì với câu trả lời đó

- Cập nhật `ProductClientController.java`:
  - Load visible reviews cho product detail
  - Kiểm tra user có thể review (đã mua + order DELIVERED)
  - Kiểm tra user đã review chưa
  - POST endpoint submit review với validation
- Cập nhật `ProductReviewService.java` thêm `getUserReviewForProduct()`
- Cập nhật `ProductReviewServiceImpl.java`:
  - Set `isVisible = false` mặc định khi tạo review
  - Implement `getUserReviewForProduct()`
- Cập nhật `product-detail.html`:
  - Tab đánh giá với average rating, review count
  - Hiển thị review form chỉ cho eligible users
  - Hiển thị thông báo khác nhau tùy theo:
    - Đã review + visible: "Cảm ơn bạn đã đánh giá!"
    - Đã review + pending: "Chờ quản trị viên duyệt"
    - Chưa mua: "Bạn cần mua sản phẩm này"
- Cập nhật `order-detail.html` link đến `/products/{id}#reviews`

### Các lỗi đã sửa

1. **Thymeleaf format error**: `#numbers.formatDecimal()` - tham số thứ 4 phải là string ('POINT') không phải số
2. **Template parsing error**: `#reviews` anchor trong URL - dùng string concatenation `+ '#reviews'`
3. **Stars display direction**: Dùng `flex-row-reverse` với CSS `peer` classes thay vì JavaScript
4. **Backend validation**: Thêm kiểm tra rating (1-5) và comment không được để trống
5. **Tab switching**: Thêm hash navigation để auto-switch khi có `#reviews`
